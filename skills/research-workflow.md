# research-workflow

## 목적

주제에 대해 체계적으로 조사하고 분석 결과를 문서화합니다.

## 사용 시점

- 새로운 기술/도구 조사
- 문제 해결 방안 탐색
- 의사결정을 위한 정보 수집
- 기존 시스템 분석

## 입력

- 조사 주제 또는 질문
- 조사 범위 (깊이, 시간 제약)
- 특별히 확인해야 할 사항

## 절차

### 1. 조사 계획

조사할 영역 정의:

```markdown
## 조사 질문
- Q1: ...
- Q2: ...

## 조사 범위
- 포함: ...
- 제외: ...

## 정보원
- 공식 문서
- 코드베이스
- 외부 자료
```

### 2. 정보 수집

각 질문에 대해:

1. 관련 정보 검색
2. 핵심 내용 추출
3. 출처 기록

### 3. 분석

수집한 정보를 바탕으로:

- 패턴 식별
- 장단점 비교
- 트레이드오프 분석
- 리스크 식별

### 4. 결론 도출

- 핵심 발견 사항
- 권장 사항 (있는 경우)
- 추가 조사 필요 영역

### 5. 문서화

`02-research.md`에 기록:

```markdown
# Research: {주제}

## 요약
핵심 발견 사항 3-5개

## 조사 내용

### {영역 1}
...

### {영역 2}
...

## 분석
장단점, 트레이드오프

## 결론
권장 사항 및 다음 단계

## 참고 자료
- [자료 1](링크)
- [자료 2](링크)
```

## 출력

- 조사 결과 문서
- 핵심 발견 사항 요약
- 권장 사항

## 출력 위치

```
projects/{project}/02-research.md
```

## 지식 추출

조사 중 발견한 재사용 가능한 지식:

- 일반적인 패턴 → `knowledge/patterns/`
- 도메인 지식 → `knowledge/domain-knowledge/`
- 유용한 참조 → `knowledge/references/`
