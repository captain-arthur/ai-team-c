## 목표

Kubernetes 클러스터의 **운영 확신(Operational Confidence)**을 제공하는 **명확하고 강력한 프레임워크**를 설계한다.

세 가지 운영 질문에 답할 수 있어야 한다:

1. "지금 클러스터가 안전한가?"
2. "내일도 안전할 것인가?"
3. "안전하지 않게 되기 전에 무엇을 해야 하는가?"

## 배경

ai-team-lab에서 개발한 모니터링 프레임워크(Block1/Block2/Block3, Operational Confidence Theory)가 존재하지만:

- 다른 사람에게 설명하기에 충분히 단순하지 않음
- 신호 간 인과관계(A→B→C→D)가 명확하지 않음
- 실제 운영 의사결정에 사용하기 어려움
- 리더십에 보고하기에 충분히 강력하지 않음

이 프로젝트는 기존 아이디어를 **정제하고 강화**하여 실용적이고 설명 가능한 프레임워크로 완성한다.

## 범위

### 포함

- 신호 계층 구조 설계
- 고장 전파 체인 모델
- 최소 신호 집합 정의
- 운영 의사결정 가이드

### 제외

- 대시보드 UI 구현
- Alert 정책 상세 설계
- Runbook 작성
