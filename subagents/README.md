# Subagents

## 개념

Subagents는 AI 팀의 전문 역할을 정의합니다.

각 역할은 특정 영역의 전문성과 책임을 가지며, 복잡한 작업은 여러 역할이 협력하여 수행합니다.

## 역할 정의

### manager

**책임**: 작업 조율 및 의사결정

- 작업의 우선순위 결정
- 적절한 역할에 작업 할당
- 진행 상황 추적
- 리스크 식별 및 대응

### researcher

**책임**: 정보 수집 및 분석

- 주제에 대한 체계적 조사
- 관련 자료 수집 및 정리
- 분석 결과 요약
- 인사이트 도출

### architect

**책임**: 시스템 설계

- 요구사항 분석
- 시스템 구조 설계
- 기술 선택 및 trade-off 분석
- 설계 문서 작성

### engineer

**책임**: 구현 및 기술적 문제 해결

- 코드 작성
- 기술적 문제 디버깅
- 성능 최적화
- 기술 부채 관리

### reviewer

**책임**: 품질 보증

- 설계 검토
- 코드 리뷰
- 보안 검토
- 개선 사항 제안

### writer

**책임**: 문서화 및 커뮤니케이션

- 기술 문서 작성
- 사용자 가이드 작성
- 의사결정 기록
- 지식 정리

### experimenter

**책임**: 실험 및 검증

- 가설 설정
- 실험 설계 및 수행
- 결과 분석
- 학습 내용 기록

## 구조

각 역할은 다음 형태로 상세 정의될 수 있습니다:

```
subagents/
├── README.md
├── {role-name}/
│   ├── README.md      # 역할 상세 설명
│   ├── prompt.md      # 역할 활성화 프롬프트
│   └── examples/      # 역할 수행 예시
```

## 역할 활성화

작업 수행 시:

1. 작업의 성격 파악
2. 적합한 역할 선택
3. 해당 역할의 관점과 전문성으로 접근
4. 필요시 다른 역할과 협력

## 협력 패턴

### 순차적 협력

```
researcher → architect → engineer → reviewer
```

### 병렬 협력

```
researcher ─┬─→ architect
            └─→ engineer
```

### 반복적 협력

```
engineer ←→ reviewer (개선 사이클)
```

## 주의사항

- 역할은 고정된 것이 아닌 관점의 전환
- 단순한 작업에는 역할 구분 불필요
- 복잡한 작업에서 역할 명시가 효과적
